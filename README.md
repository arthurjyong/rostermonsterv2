# Roster Monster v2

Roster Monster v2 is a planning-stage project to rebuild roster allocation as a reusable core with department-specific template packs. Google Sheets will remain the operational front-end, while the allocation logic is designed to be reusable across departments. The first department implementation target is CGH ICU/HD.

This repository is for architecture and contract planning before implementation begins. Current status: **planning / architecture phase**.

## Core documents
- `docs/blueprint.md` — Master planning outline for scope, boundaries, and unresolved architecture questions.
- `docs/template_contract.md` — Conceptual contract for how each department template defines roster behavior.
- `docs/domain_model.md` — Draft normalized domain model outline for future internal data structures.
- `docs/roadmap.md` — Phased implementation sequence from documentation through integration and hardening.
- `docs/decision_log.md` — Living record of key product and architecture decisions.
